4. Environment & Config Blockers

Two .env.local items must be corrected before real customers pay.

Blocker 4.1 · Webhook secret is a placeholder.
RAZORPAY_WEBHOOK_SECRET=your_razorpay_webhook_secret — the webhook will return 503 until this is set. Without it, credits only apply via the client-side verify step. If the user closes their browser before verify runs, the payment is captured but credits are not granted — they've paid you with nothing to show.

Fix: Razorpay dashboard → Settings → Webhooks → create webhook pointing to https://kunjaraos.com/api/billing/webhook with event payment.captured. Copy the generated secret into .env.local and into Vercel's env vars.
Blocker 4.2 · Razorpay keys are LIVE, not test.
RAZORPAY_KEY_ID=rzp_live_SeeYZWNmVe55zF. Every dev test charges real money. Smoke tests today created a real live order for ₹1,999 (it auto-cancels unpaid so no harm done, but the risk is real).

Fix: Generate test keys in Razorpay dashboard. Use rzp_test_… in local dev and staging; keep rzp_live_… only in the production Vercel env.

6. How to Execute — The Playbook

6.1 · Rules we learned this sprint

  • One plan at a time. The codebase already shows what happens when UI ships ahead of backend — lib/api.ts accumulated 15+ endpoint references to nothing. Build backend first, UI second.
  • Verify schemas before writing persistence. The credits_added vs events_used model mismatch happened because two developers wrote to the same table with different assumptions. Check Supabase dashboard first.
  • Delete orphan endpoints aggressively. Three insecure routes (add-credits, verify-payment, capture-payment) survived because nothing called them. Any unused endpoint is a liability.
  • Hardcode Supabase creds in the client lib, not env vars. The repeated "fix supabase env crash" commits in git history prove process.env reads at module-load break on Vercel cold starts. String literals in lib/supabase/client.ts + server.ts side-step this.

  • Ask for plan first when the task touches 3+ files. Plan mode catches the kind of mismatch that caused the billing/usage schema drift.
  • Launch a test user to click through after each Claude session. Unauth smoke tests caught the ones this sprint didn't catch.
  • Commit in small scopes. Three commits today, each with a focused message and a concrete Before/After. Makes rollback trivial.
